# 개인 다이어리 웹 애플리케이션 요구 사항 문서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 개인 다이어리 웹 애플리케이션 |
| **목적** | 개인이 일상을 기록할 수 있는 다이어리 웹 애플리케이션 제공 |
| **대상 사용자** | 일상 기록이 필요한 개인 사용자 |

---

## 2. 핵심 기능 요구 사항

### 2.1 일기 작성

- **날짜 선택**: 사용자가 기록할 날짜를 선택할 수 있어야 한다.
- **제목 입력**: 해당 날짜의 일기 제목을 입력할 수 있어야 한다.
- **본문 입력**: 해당 날짜의 일기 내용(텍스트)을 입력할 수 있어야 한다.

### 2.2 시각적 구분 (날짜별 테마)

- **날짜 기반 색상 생성**: 날짜 값(`yyyyMMdd`)을 기준으로 배경 색상과 텍스트 색상이 자동 생성되어야 한다.
- **색상 조화**: 배경 컬러와 텍스트 컬러는 서로 어울리도록 생성되어야 한다.
- **일관성**: 동일한 `yyyyMMdd` 값에는 항상 동일한 색 조합이 적용되어야 한다.

### 2.3 데이터 저장

- **저장 형식**: 일기 데이터는 **TSV(탭으로 구분된 값)** 파일로 저장한다.
- **저장 단위**: 일기 목록 또는 단일/복수 일기 항목이 TSV 구조에 맞게 저장되어야 한다.

### 2.4 TSV 파일 내보내기

- **내보내기 기능**: 저장된 일기 데이터를 TSV 파일로 내보낼 수 있어야 한다.
- **다운로드**: 사용자가 요청 시 현재 저장된 전체 일기 목록을 TSV 형식으로 다운로드할 수 있어야 한다.
- **형식 일치**: 내보내기 파일은 [3. 데이터 저장 규격 (TSV)](#3-데이터-저장-규격-tsv)과 동일한 규격(탭 구분, UTF-8 등)을 따른다.
- **파일명**: 내보내기 파일명은 사용자가 인지하기 쉬운 형식(예: `diary_yyyyMMdd.tsv` 또는 `diary_export.tsv`)으로 제공한다.

---

## 3. 데이터 저장 규격 (TSV)

- 파일 확장자: `.tsv`
- 구분자: 탭(`\t`)
- 저장 시 고려 항목:
  - 날짜(`yyyyMMdd` 또는 `yyyy-MM-dd` 등 정의된 형식)
  - 제목
  - 본문 텍스트
- 인코딩: UTF-8 권장

---

## 4. 비기능 요구 사항 (권장)

- **반응형**: 다양한 화면 크기에서 사용 가능하도록 구성한다.
- **접근성**: 키보드/스크린 리더 등 기본 접근성을 고려한다.
- **성능**: 날짜 선택·저장·로드 시 사용자 대기 시간을 최소화한다.
